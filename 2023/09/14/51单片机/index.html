Delay延时函数

延时函数的目的在于等一段时间后再执行接下来的代码。
一般在按键、防io抖动、计时、任务调度中使用。

循环延时

时钟周期 = 1 / 晶振频率
状态周期 = 2 * 时钟周期
机器周期 = 完成一个基本操作所需要的时间 = 时钟周期 * n（n取决于单片机型号）
指令周期 = 完成一次指令所需的时间 = 一或者两个机器周期

计算公式：

延时时间 =（count * 指令周期所需的时钟周期数 * 循环中的指令数）/ 时钟频率

例如：

晶振频率：8Mhz
机器周期：2个时钟周期
指令周期：6个机器周期 = 12个时钟周期

延时时间 = 50 * 12 * 2 * (1 / 8M) = 0.000075 s = 75 us

#include <REGX51.H>

void Delay1ms(unsigned int xms)
unsigned char i, j;
while(xms)
{
i = 2;
j = 239;
do
{
while (–j);
} while (–i);
xms–;
}
}

void main()
{
while(1)
{
P2=0xFE;//1111 1110
Delay1ms(500);
P2=0xFD;//1111 1101
Delay1ms(500);
P2=0xFB;//1111 1011
Delay1ms(500);
P2=0xF7;//1111 0111
Delay1ms(500);
P2=0xEF;//1110 1111
Delay1ms(500);
P2=0xDF;//1101 1111
Delay1ms(500);
P2=0xBF;//1011 1111
Delay1ms(500);
P2=0x7F;//0111 1111
Delay1ms(500);
}
}

STC89C52的T0和T1均有四种工作模式：
模式0：13位定时器/计数器
模式1：16位定时器/计数器（常用）
模式2：8位自动重装模式
模式3：两个8位计数器

时钟
计数系统
16位计数器-能计两个字节
TL0-time or low -低字节
TH0-time or hight -高字节
0-定时性为0
总共可以存储65535个数 0-65535 每隔1us计数加1 总共定时时间65535us 65毫秒
64535 离计数器溢出差值1000，所以计时时间为1us
脉冲 计数器+1 65535 溢出 返回0-制一个标志位 TF0(time of flag) 向中断系统申请中断
控制位
TR0 控制定时器启动暂停

中断系统
使CPU具有对外界紧急事件的处理能力
计数器溢出相当于中断源
CPU总是先响应优先级最高的中断请求
中断嵌套
CPU暂停对原来中断源的服务程序，转去处理优先级更高的中断请求源，处理完以后，再回到原低级中断服务程序
含有中断嵌套的中断系统称为多级中断系统，没有中断嵌套功能的中断系统称为单级中断系统

STC89C52中断资源
中断的资源和单片机的型号是关联在一起的，不同的型号可能会有不同的中断资源
中断源个数：8个（外部中断0，定时器0中断，外部中断1，定时器1中断，串口中断，定时器中断2，外部中断2，外部中断3）

定时器相关寄存器
单片机通过C语言配置寄存器来控制内部线路的连接
寄存器是连接软硬件的媒介
在单片机中寄存器就是一段特殊的RAM存储器，一方面，寄存器可以存储和读取数据，另一方面，
每个寄存器背后都连接了一根导线，控制着电路的连接方式
寄存器相当于一个复杂机器的“操作按钮”
